## 🔴 根因一：日干宫绑定全天 · 所有比赛共享同一基线

### 问题
同一日所有比赛的奇门排盘中，`日干`是当天日柱的天干——**同一天4场比赛的日干落宫完全相同**。

```
6月21日·日柱 = 丙申日 → 日干=丙 → GAN_JI_GONG['丙'] = '巳'
→ 日干在地盘的宫位：巽4宫（木）

4场比赛的日干宫全部是 巽4 → 基线完全相同！
```

**这意味着：** 奇门遁甲对同一天不同比赛的区分度，仅靠 `时干` 落宫来体现——而时干的变化范围和区分度极其有限。

### 影响
- 荷兰vs瑞典（01:00 乙丑时）→ 时干乙=离9(火) → **时克日**→判客胜 ❌（实际荷兰5-1）
- 德国vs科特迪瓦（04:00 丙寅时）→ 时干丙=巽4(木) → 日时同宫？→判**日克时**→主胜 ✅
- 突尼斯vs日本（12:00 庚午时）→ 时干庚=兑7(金) → **时克日**→判客胜 ✅（实际日本4-0）

**结论：** 日干固定导致同一天的预测高度同质化，时干的区分不足以反映真实的实力差异。

## 🔴 根因二：六壬100%失败 · 判平局成系统性偏差

### 4场比赛六壬全部判错，其中3场误判平局

根本问题出在三传解读逻辑上：

```javascript
// divination.js 中的平局判定逻辑过于宽松
const lFinal2 = liuren.secondHalf;
const lLabel = lFinal2.includes('主队') ? 'home' :
                lFinal2.includes('客队') ? 'away' : 'draw';
```

关键在于 `secondHalf` 字段的计算：末传与日干**比和**时→判"终局均势"→映射为 `draw`。

而实际上：
- **末传比和 ≠ 平局**。末传五行与日干比和只是表示终局力量均衡，但在足球比赛中，终局均衡可以对应多种结果：大胜后的垃圾时间、双方都放弃进攻、或者真正的平局。
- 六壬模型中，只要末传不生也不克日干，几乎全判"均势/平局"，导致平局输出概率远高于实际情况。

### 具体验证
| 比赛 | 三传 | 末传与日干关系 | 六壬判 | 实际 |
|:--|:--|:--|:--|:--|
| 荷兰vs瑞典 | 巳→亥→巳 | 比和→均势 | 平局1-1 | 荷兰5-1 |
| 德国vs科特迪瓦 | 未→子→巳 | 比和→均势 | 平局1-1 | 德国2-1 |
| 厄瓜多尔vs库拉索 | 申→亥→寅 | 生→优势 | 主胜2-1 | 0-0平局 |
| 突尼斯vs日本 | 辰→巳→午 | 比和→均势 | 平局1-1 | 日本4-0 |

**3场判平局全错（实际都是大比分胜负），1场判主胜也错（实际平局）。**

## 🔴 根因三：比分预测完全硬编码 · 毫无统计学基础

```javascript
// divination.js 中的比分预测逻辑
if (qLabel === 'home') { qScore = qimen.shengKe === '日克时' ? '2-1' : '2-0'; }
else if (qLabel === 'away') { qScore = qimen.shengKe === '时克日' ? '1-2' : '0-2'; }
else { qScore = '1-1'; }
```

**问题：**
- 所有"日克时"的比赛 → 一律判 `2-1`，不管实际实力差距
- 所有"时克日"的比赛 → 一律判 `1-2`
- 平局 → 一律 `1-1`
- 总进球数被锁定在 1-3 球之间

| 德国 2-1 科特迪瓦 | 2-3球 | **3球** | ✅ 巧合命中 |

**4场只有1场的总进球在预测范围内 → 比分预测精度 ≈ 0%，纯粹靠蒙。**

## 🔴 根因四：术数模型对足球比赛本质的认知错位

### 核心假设错误
代码将 `日干宫 = 主队(左队)`、`时干宫 = 客队(右队)` 作为基本框架。

但在奇门遁甲的传统用法中：
- **日干**代表问事人（占卜者本人/求测方）
- **时干**代表所问之事（事情本身/对方）

将日干映射为主队、时干映射为客队——这个映射本身没有经典依据，是拍脑袋造的。

更合理的做法可能是：
- 以**比赛时间**排盘，看时干宫的吉凶状态
- 以**双方队名/方位**取用神
- 引入**实力差距**作为先验概率修正

### 缺乏先验概率
统计模型的ELO+泊松可以给出**荷兰胜率65%+**，而奇门模型完全无视这个信息——"时克日→客胜"覆盖了ELO信号。这是**纯术数 vs 纯统计的信息孤岛问题**。

## 🔴 根因五：节气判定错误 · 排盘基础不牢

当前代码中6月21日用的是**芒种→阳遁6局**，但实际检查：
- 芒种：约6月5-6日
- 夏至：约6月21-22日

6月21日的节气判定需要精确到时分。如果比赛时间在夏至交节**之后**，应该是**阴遁**而非阳遁，整盘都会翻转。

当前代码使用 `getPrevJieQi()` 取上一个节气，但未检查是否已过夏至交节时间点。6月21日靠近夏至分界线，这可能导致排盘方向（阳遁/阴遁）完全错误。

## 🎯 改进方向

### 短期修复
1. **日干不应绑定日柱天干** — 改为每场比赛独立取用神，用比赛时间+双方队名/首字笔画/方位取奇门用神
2. **比分预测引入ELO先验** — 术数只输出胜负倾向（概率），具体比分由泊松模型计算
3. **六壬平局判定加严** — 末传比和不等于平局，需要结合课体（如伏吟/反吟/八专等）综合判断
4. **节气判定加校验** — 比赛时间与节气交节时刻的时分比较，确保阴阳遁方向正确

### 长期架构
5. **术数输出"倾向概率"而非硬判决** — 如"主胜倾向60%/平局25%/客胜15%"，与统计模型做贝叶斯融合
6. **反馈闭环** — 每轮比赛后自动对比预测vs真实，更新模型权重
7. **弃用纯六壬比分预测** — 大六壬更适合定性（吉凶趋势）而非定量（具体比分），应降权或仅作辅助参考

## 📊 总结

| 维度 | 结论 |
|:--|:--|
| **奇门遁甲** | 胜负方向命中率50%，有一定参考价值但远不及统计模型 |
| **大六壬** | 当前实现完全不可用（0%准确率），平局偏差是系统性问题 |
| **综合判决** | 奇门主导+六壬无效 = 与纯奇门无异 |
| **比分预测** | 硬编码导致完全不可信，需要引入实际统计模型 |
| **排盘精度** | 节气判定存在边界bug，需修复 |
| **最核心问题** | 日干联动全天导致区分度不足 + 六壬平局泛滥 |

> **底线判断：** 当前术数模块在足球预测上的准确率（50%胜负方向+0%比分精度）远低于统计模型（ELO+泊松），暂时不具备独立预测能力。建议在代码中标注"仅作趣味参考"，并在每次赛后自动运行对比、持续追踪。
